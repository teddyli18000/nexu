# コントリビューター報酬＆サポート

**nexu** に貢献していただく際、「PR がマージされた」だけでなく、以下のことも大切にしています：

- あなたの真剣な取り組みが見える形で残ること
- 貢献がマージ履歴だけに留まらないこと
- 成長が継続的に記録・蓄積されること
- 初めての参加でも、コミュニティの歓迎を感じられること

## 私たちが提供するサポート

`good-first-issue` と初回貢献者に対して、できる限り以下のサポートを提供します：

- **より明確なタスク範囲**：小さく明確で検証可能なタスクに分割
- **より具体的なレビューフィードバック**：変更が必要な場合、理由と方向性を説明
- **初回貢献ガイダンス**：初参加時に必要な説明とアドバイスを提供
- **公開感謝とポイント記録**：マージ後、貢献者感謝・ポイントフローに入ります

まだ始めていない方は、こちらをご覧ください：

- [Good First Issue コントリビューターガイド](/ja/guide/first-pr)
- [コントリビュート](/ja/guide/contributing)
- [Good First Issue リスト](https://github.com/nexu-io/nexu/labels/good-first-issue)

## 貢献の表示場所

貢献がマージされると、以下の場所に表示されます：

1. **GitHub README**：コントリビューター表示、**日次・月次・累計リーダーボード** を同期表示
2. **Discord `#leaderboard` チャンネル**：Bot がリーダーボードを自動発信

## 現在のポイントルール

以下は現在の基準です。チームが調整を発表した場合は、最新版が優先されます：

| 貢献タイプ | ポイント | 説明 |
| --- | --- | --- |
| PR マージ（starter 難度） | 10 | typo 修正、設定調整 |
| PR マージ（easy 難度） | 25 | バグ修正、小機能 |
| PR マージ（medium 難度） | 50 | 完全な機能実装 |
| PR マージ（advanced 難度） | 100 | アーキテクチャレベルの変更 |
| Skill を ClawHub に公開 | 80 | 独立実行可能な Skill |
| Bug Report（確認済み） | 10 | 再現手順付きのバグ報告 |
| ドキュメント | 15-30 | 篇幅と深さによる |
| 翻訳 | 20 / 言語 | 各言語パック |
| Code Review（採用） | 15 | 実質的な提案のある Review |
| Discussion 質の高い回答 | 10 | Answer としてマークされた返信 |
| 新規コントリビューター紹介 | 20 | 紹介された方の初 PR マージ後に紹介者へ |

## ポイントの意味

ポイントは「ゲーミフィケーションでのスコア稼ぎ」ではなく、価値ある貢献を一つひとつ真剣に記録するためのものです。

- 小さな修正も価値がある
- ドキュメント、翻訳、Review が見過ごされなくなる
- 初回貢献者がより明確なポジティブフィードバックを得られる
- 長期貢献者がコミュニティで可視的なアイデンティティを築ける
- ポイントはコミュニティバッジに交換可能
- ポイントはオープンソース貢献の履歴として蓄積される

## 最も簡単なタスクから始めたい場合

**Good First Issue** コントリビューターを継続的に募集しています。

これらのタスクは通常スコープが小さく、方向がより絞られており、初回貢献に適しています。

タスクを選ぶ：[Good First Issue リスト](https://github.com/nexu-io/nexu/labels/good-first-issue)
