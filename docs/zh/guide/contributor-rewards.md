# 贡献奖励与支持

如果你为 **nexu** 做出贡献，我们希望这件事不只是“PR merge 了”，还包括：

- 你的改动被及时 review
- 你的贡献被公开看见
- 你的投入有明确积分与奖励
- 你在第一次贡献时能拿到必要帮助

## 我们会提供什么帮助

对于 `good-first-issue` 和首次贡献者，我们会尽量提供这些支持：

- **更清晰的题目范围**：尽量把题目拆成单一方向、可自测、可验收的小任务
- **公开 review 节奏**：通常以 **约 48 小时内首次 Review** 为目标，但仍以当时维护者人力为准
- **协作式反馈**：如果 PR 需要修改，我们会尽量指出原因和方向，而不是只给结论
- **首次贡献引导**：第一次贡献者在 PR merge 后，会收到后续参与建议和下一步推荐
- **公开致谢**：合入后会进入贡献者致谢与积分流程

如果你还没开始，建议先看：

- [第一次提 PR 指南](/zh/guide/first-pr)
- [参与贡献](/zh/guide/contributing)
- [Good First Issue 列表](https://github.com/nexu-io/nexu/labels/good-first-issue)

## 你的贡献会展示在哪里

如果你的贡献被合入，后续会在这些位置被看到：

1. **GitHub README**：动态展示 Top 10，包含 **月度榜** 与 **历史累计榜**
2. **Discord `#leaderboard` 频道**：Bot 每月自动发布排行榜

这意味着你的贡献不是“默默合进去了就结束”，而是会被持续看见、持续累计。

## 当前积分规则

以下为当前口径；若后续团队公告调整，以最新公示为准：

| 贡献类型 | 积分 | 说明 |
| --- | --- | --- |
| PR 合入（starter 难度） | 10 | 修复 typo、配置调整 |
| PR 合入（easy 难度） | 25 | Bug 修复、小功能 |
| PR 合入（medium 难度） | 50 | 完整功能实现 |
| PR 合入（advanced 难度） | 100 | 架构级改动 |
| Skill 发布到 ClawHub | 80 | 独立可运行的 Skill |
| Bug Report（被确认） | 10 | 提供复现步骤的 Bug 报告 |
| 文档贡献 | 15-30 | 按篇幅和深度 |
| 翻译贡献 | 20 / 语言 | 每个语言包 |
| Code Review（被采纳） | 15 | 有实质性建议的 Review |
| Discussion 优质回答 | 10 | 被标记为 Answer 的回复 |
| 社区推荐新贡献者 | 20 | 推荐人首次 PR 合入后给推荐者 |

## 这些积分意味着什么

这套规则不是为了“游戏化刷分”，而是为了让每一种对社区有价值的贡献，都能被认真记录下来。

你可以把它理解成一种持续积累的开源贡献履历：

- 小修小补也有价值
- 文档、翻译、Review 不再被忽略
- 首次贡献者能更明确看到正反馈
- 长期贡献者能在社区里积累可见身份

## 如果你想从最容易开始的题目切入

我们正在持续寻找 **Good First Issue** 贡献者。

这类题通常更适合第一次贡献，因为它们往往具备这些特点：

- 范围更小
- 改动更聚焦
- 验收更明确
- review 更容易推进

直接去挑题：

- [Good First Issue 列表](https://github.com/nexu-io/nexu/labels/good-first-issue)
